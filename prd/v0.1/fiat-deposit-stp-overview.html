法币入金自动化 STP

将 100% 人工处理的法币入金，通过直通式处理（Straight-Through Processing）实现 80%+ 的全自动入账

MetaComp Pte. Ltd. MAS MPI + DPT 版本 v1.12 2026-06-04
背景与痛点
当前入金完全依赖人工，效率低下、易出错且无法规模化。

⚠️ 当前问题

  • 资金到账后，Lark 收到 webhook 通知，但后续全靠人工操作
  • Ops 需手动匹配客户账户、补全银行信息、点击入账——每笔耗时数小时
  • COBO 预审查表单收集的是客户自填数据，而非实际到账交易信息，存在合规盲点
  • 实际到账信息与预填表单不一致时，风控筛查无法检测差异
  • 无法规模化：随业务增长，人力瓶颈将成为关键风险

✅ STP 方案

  • 银行 webhook → 自动客户识别（VA 匹配 / 参考码匹配）
  • 自动判断资金性质（本人 / 第三方），基于模糊名称匹配
  • 自动触发 VisionX 合规风控筛查，使用真实到账交易数据
  • 筛查通过后，按 Value Date 自动入账，无需人工干预
  • 无法识别的资金进入待认领账本，通过 Task Center 推送给 Ops 处理
  • 彻底取消 COBO 预审表单（已获 Compliance 确认）

三大核心变化
相比现有流程的关键差异

取消 COBO 预审表单

COBO 表单收集的是客户自填声明，而非真实到账数据。筛查结果缺乏可信度，且资金到账时系统复用旧表单数据，无法检测实际付款方差异。

变化：Compliance 已确认取消。VisionX 筛查改用银行 API 返回的真实交易数据，100% 覆盖。

法币账户映射参考表

新建统一的账户映射表，覆盖 GLDB、SGB、TransferMate、Tazapay 等所有渠道。客户开设 VA 或生成参考码时自动写入，无需 Ops 手动维护大部分记录。

价值：提供账号→客户的确定性映射，是 VA 匹配和账户状态检查的数据基础。

Ops 工作流重构

原有手工处理路径被 Task Center 替代。Ops 只需处理规则引擎无法自动解决的少数异常：未识别资金、账户状态异常、模糊分类确认。

变化：每条任务预填所有可用信息，候选客户列表自动生成，Ops 点选确认即可，无需从头查找。

姓名模糊匹配算法
NameMatchService 是独立模块，返回归一化匹配分 [0.0, 1.0]，供规则引擎决策分类阈值。

🔤 处理能力

  • 大小写、多余空格、标点符号归一化
  • 特殊字符 ASCII 转写：José → Jose，Müller → Mueller，François → Francois
  • 中文姓名拼音转换：「张三」↔ "Zhang San" / "Zhangsan"
  • 企业后缀分离："Pte Ltd"、"Inc"、"GmbH" 等不影响核心名称匹配
  • Token 顺序无关匹配："Lim John" 与 "John Lim" 得分相同
  • 拼音结构校验：防止"王小华"与"Hua Xiao Wang"误判为同一人

📊 分类阈值（法币入金场景）

  • S ≥ 0.85 → 本人（1st Party）：必填字段较少
  • 0.5 ≤ S < 0.85 → 模糊区间：路由到 Task Center，Ops 人工确认
  • S < 0.5 → 第三方（3rd Party）：需额外提供付款用途

阈值存于数据库配置，无需重新部署即可调整。算法完全确定性，每次匹配结果可追溯至具体 token 对比步骤，满足监管审计要求。

各方影响
对主要干系人的具体改变

🏢 Operations 团队

从逐笔手工处理变为异常处理。Task Center 预填所有可用信息，候选客户自动排序，确认操作一键完成。预计效率提升 4 倍。

⚖️ Compliance / TxM

筛查改用真实到账数据，消除旧版 COBO 表单与实际交易不一致的合规盲点。筛查自动触发，无需手动发起。VisionX 整合由 VisionX 团队负责。

💰 Finance / Treasury

Value Date 逻辑与银行结单保持一致，保证资金安全储存时机准确。EOD 对账纳入待认领账本余额，确保账本平衡。

👤 客户 / End User

自动入账从 4–24 小时缩短至 1 分钟以内。资金到账后收到邮件/Webhook 通知。缺少字段时通过 Task Center 补充，无需联系客服。

🔌 合作方 / Partner API

新增 Missing Fields Response 接口，可通过 API 实时获知需补充哪些字段并回传。支持预存已知付款人信息，提升自动识别率。

🔧 Engineering

规则引擎与渠道配置分离，新增渠道无需修改核心逻辑。每条规则独立可测试。NameMatchService 为纯函数模块，可复用于提现、转账等场景。
