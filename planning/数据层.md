# 数据层 — 架构与工作准则

## 一句话

前端所有数据走 `/api/v1/*`，DEV 下 MSW Service Worker 拦截并返回 seed 数据，Prod 下命中真后端。页面代码与数据源解耦。

---

## 分层架构

```
src/
  types/               ← 前端 domain model（TypeScript interface）
                          与后端 response shape 解耦，前端唯一真理
  services/
    _client.ts         ← apiFetch() + ApiClientError + 重试
    index.ts           ← 统一导出
    hooks/useXxx.ts    ← TanStack Query hooks（页面唯一数据入口）
    helpers/           ← 纯函数（金额格式化、日期等）
  mocks/
    seed/              ← typed seed 数据，一表一文件
    handlers/          ← MSW HTTP handler，一模块一文件
    browser.ts         ← setupWorker() + HMR
```

适配器层 `src/adapters/`：**暂缓**。当前 seed 数据直接对齐 `src/types/`。后端 shape 与前端 model 出现结构性差异时再加这一层做转换，不提前引入。

---

## 铁律

### 1. 数据唯一来源在 `src/mocks/seed/`
页面禁止内联 `const MOCK_*` / `const SAMPLE_*` 业务数组。UI 配置（列定义、tab 列表等）可以内联。

### 2. 金额用 minor units（整数）
`usd(384721)` = $3,847.21（cents）；`btc(50000000)` = 0.5 BTC（satoshi）。永远不存格式化字符串。

### 3. 时间用 ISO 8601
`'2026-05-11T10:30:00Z'`。seed 数据用相对时间 helper `iso({ daysAgo: N })` 生成，不会过时。

### 4. ID 用可读前缀
`org_alibaba`、`user_001`、`wallet_ali_usd`——不用 UUID。mock 调试友好。

### 5. Service hooks 用 TanStack Query
`staleTime: 5min`、`retry: 1`、`refetchOnWindowFocus: false`（QueryClient 全局默认已设）。Hook 返回值暴露 `{ data, isLoading, isError, error }`，页面不直接调 apiFetch。

### 6. 响应 envelope 格式
所有 handler 返回 `{ data: T }` 或 `{ error: { code, message, detail? } }`。`apiFetch` 拆包，调用方拿到的是 `T`，不感知 envelope。

## 新加一个 API 的标准流程

```
1. src/types/<entity>.ts      定义 domain interface
2. src/mocks/seed/<entity>.ts 写 seed 数据（typed array）
3. src/mocks/handlers/<entity>.ts  注册 MSW handler
4. src/mocks/handlers/index.ts     把 handler 加进 handlers 数组
5. src/services/hooks/useXxx.ts    写 TanStack Query hook
6. src/services/hooks/index.ts     导出 hook
```
